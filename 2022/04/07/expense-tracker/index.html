老爸的私房錢 Expense-Tracker

Created2022-04-07|Updated2022-06-04|Project Review
|Word count:818|Reading time:3min|Comments:

第一個從無到有親手打造的專案，對自己來說是個挑戰，也是一個能夠統整並複習這陣子所學的專案，除了這陣子常用的資料庫操作、路由設計，也重新喚醒了自己前幾個學期的切版技能並搭配 handlebars helper 去強化前端頁面。

Live Demo | Github

使用的技術

  • Bootstrap 5 為前端框架
  • Handlebars 為前端樣板引擎
  • Node.js 的 Express 為後端框架
  • 使用 NoSQL 資料庫 (MongoDB)
  • 使用 ODM (Mongoose) 直接以 JavaScript 語法操作 DataBase

專案細節

哪部分相對能掌握？

資料庫 Create/Read/Update/Delete 相較其他部分較能掌握，以前都是以 Promise 語法撰寫，這次嘗試著以 async/await 去撰寫。

哪裡花了最多時間？

頁面切版的部分花了比預期久的時間去嘗試出自己要的感覺，登入驗證的需要透過第三方套件以及相關程式的認證，這部分也花了一些時間。

過程中碰到什麼困難？如何克服？

這次是自己親手，從無到有打造一個新的專案，面臨到的技術也相對來得多，有許多不太熟悉的技術也透過這次操作更加得心應手。

學習到透過 handlebars helper 讓支出的 categoryId 與 Model Category _id 去相互比較，並針對比較結果去設定前端樣板內 select 和 option 顯示的結果。

過程中對哪個技術有特別深刻的學習？

主要是 mongoose 的 populate 的功能。

原先一開始是採用 MongoDB 的 $lookup aggregate 去查詢相關聯的兩個 Models。

由於 aggregate 算是高級查詢，針對該專案使用的關聯查詢不需要到用到較為複雜的查詢概念，僅以 mongoose 的 populate 即可完成相關聯查詢的需求。

雖然之後改用 populate 去引用其他相關聯的 Model，不過一開始在 populate() 內還以為要填寫另一個關聯的 Model，後來才發現是填寫查找 Model 內的與另一個 Model 有相關聯的欄位。

額外新增功能

  • 分類的 CRUD：刪除的部分採用的是當該分類被刪除時，相關的支出會一併被刪除的設置。
  • 日期 filter：僅針對年份進行篩選功能。
  • 重設密碼 (Nodemailer)：透過 nodemailer 的功能，以自己的個人 email 發送驗證碼，讓忘記密碼的使用者完成重設密碼。
