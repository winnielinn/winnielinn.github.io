變數宣告 var / let / const 到底差在哪

Created2021-10-06|Updated2022-06-10|JavaScript
|Word count:1.6k|Reading time:5min|Comments:

在 coding 的日常中，宣告變數這種最基本的起手式，像我這樣的轉職仔在找資料的時候，一定很常看到var、let、const這宣告變數的三劍客。今天就來分享一下這三劍客在使用方式和形態上有什麼差異。

Var

在 2015 (ES6) 以前，宣告變數僅能以 var 來宣告。

特性

  • 作用域

以 var 宣告的變數，其作用域僅該在函式內。

但是如果在函式外宣告，其作用域則為全域 (Global)。

這是因 var 沒有區域作用域的概念，此時的 i 為全域變數，當每次迴圈 i + 1 時，i 會被重新賦值。當 i === 4 時會跳出迴圈，程式保存了最後的 i 值，並在迴圈結束時印出最後的值 (跳出迴圈時的 i 值) 。

所以使用 var 在區域語句 (while/if else/for…) 宣告變數時，會污染全域變數，導致程式碼維護難度增加。

  • 重複宣告

當使用 var 宣告一個已經被宣告過的 var 變數時，這個宣告會被忽略。

在第一行用 var 宣告了 print 這個變數並賦值，而第二行隨即又執行了一模一樣的動作，只差在賦予的值不同。

這邊程式並不會報錯的原因是，第二行的 var print 的宣告會被忽略，但是 print = '第二次宣告' 的動作還是會執行。

所以你可以把以上的程式碼看成：

  • 提升 (Hoisting)

var 還有一個神奇的特性是，可以先宣告再使用。

不論你在位於程式碼的哪一行用 var 宣告變數，一律視為在函數的第一行宣告，這是因為宣告都會被提升到最頂端的關係。

由於變數提升的特性，第一個印出來的 temp 會是 undefiend 而不是 ReferenceError，這邊要特別注意的是，只有「宣告」這個動作有 hoisting 的特性，賦值 (把值指定給變數) 的動作不會 hoisting。

Let

在 ES6 以後，新增了 let 和 const，讓區塊作用域 (block scope) 的觀念更為明確，也讓 var 漸漸被 let 和 cost 所取代。

特性

  • 作用域

宣告的變數僅在該區域有效，也就是 { } 包住的區域，一但離開 { } 範圍，這個變數就不會被存取到。

由於 i 值只活在 for 迴圈這個區域，而不存在於全域，因此在最後結束 for 迴圈印出全域的 i 值時，才會跳出 ReferenceError。

可以有效地避免用 var 宣告的區域變數會污染全域函數的問題。

  • 不允許重複宣告

於同一個 Block 無法重複宣告該變數。

但可以不斷賦值。

如果 let 宣告的變數在不同層的 Block，就不會報錯。

  • 宣告的變數不提升

let 宣告的變數並沒有提升的特性。

因此當第一行先印出 temp 的時候，程式會直接報錯，而不是印出 undefiend。

Const

const 可以視為 let 的加強版，基本上擁有 let 的所有特性，但仍有些許不同。

特性

  • 宣告時必定要賦值

const 在宣告變數時就會進行初始化，無法等到之後再賦予值。因此必定要在一開始就給予值作宣告，否則將會報錯。

  • 不允許重複賦值

使用 const 宣告變數後，變數的值是固定的，無法透過重複賦值來更改。

  • 可以新增和修改物件或陣列內的值

const 雖然常用來宣告不能被更動的值，不過若 const 宣告的變數是物件或是陣列是可以被修改及刪減的，但一樣無法被重新賦值。

如果想讓 const 宣告的變數完全無法被更改，可以使用 Object.freeze() 來凍結該變數。

如此一來，就可以讓 const 宣告的變數被凍結住，無法修改值，也無法增加值。

總結

最後的最後，來總結一下這三劍客的特性：

  • var 是 ES6 以前宣告變數的方式，宣告變數的 scope 為函式作用域，若在函式外宣告，作用域則為全域，這讓以 var 宣告的區域變數容易污染全域變數，且帶有提升的特性，並可以不斷重複宣告變數。

  • let 和 const 宣告的變數的 scope 則為區域作用域，且禁止在同一層 Block 重複宣告變數，兩者的差別在於使用 let 宣告的變數一開始不一定要賦值，且可以被重新指定值；而 const 宣告的變數在一開始就需要賦值，且不可以被重新指定值。
